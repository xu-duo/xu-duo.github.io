图解 HTTP 读书笔记

2017-02-20

TCP/IP

1、TCP/IP 是互联网相关的各类协议族的总称。HTTP 为其内部的一个子集。

2、TCP/IP 分为四个层：应用层、传输层、网络层、数据链路层；分层的好处：当某个地方改变设计时，只需把变动的层替换即可。

  • 应用层：决定向用户提供应用时通信的活动。其中，FTP（File Transfer Protocol，文件传输协议）和 DNS（Domain Name System，域名系统）是其中两类，HTTP 协议也处于该层。

  • 传输层：提供处于网络连接中的两台计算机之间的数据传输。包括两个性质不同的协议，TCP（Transmission Control Protocol，传输控制协议）和 UDP（User Data Protocol，用户数据报协议）。

  • 网络层：又名网络互连层。是用来处理在网络上流动的数据包（数据包是网络传输的最小数据单位），该层规定了传输路径。

  • 链路层：又名数据链路层，网络接口层。用来处理网络的硬件部分，包括控制操作系统、硬件的设备驱动、NIC（Network Interface Card，网络适配器，即显卡），及光纤及物理可见部分。

3、与 HTTP 相关的协议：IP、TCP、DNS

  • IP 协议：Internet Protocol 网际协议，位于网络层，负责传输，把各种数据包传送给对方。两个重要的条件：IP 地址和 MAC（Media Access Control Address）地址。IP 地址指明节点被分配地址、MAC 地址指网卡所属的固定地址。IP 地址和 MAC 地址之间的通信使用 ARP（Address Resolution Protocol）协议，ARP 是一种用以解析地址的协议，根据通信方的 IP 地址就可以反查出对应的 MAC 地址。

  • TCP 协议：位于传输层，提供字节流服务（为了传输方便，将大块数据分割成报文段为单位的数据包进行管理）。TCP 协议采用了三次握手协议：

  • DNS 服务：位于应用层，提供域名到 IP 地址之间的解析服务。

HTTP

1、基本概念

  • 请求报文：请求方法（GET、POST等）、请求 URI、版本协议、可选的请求首部字段和内容实体构成。（如：POST /form/entry HTTP/1.1）

  • 响应报文：协议版本、状态码、用以解释状态码的原因短语、可选的响应首部字段以及实体主体构成。

  • HTTP 协议是无状态协议，即对于发送过的请求或响应不做持久化处理；为了实现期望的保持状态功能，引入 Cookie 技术。

2、HTTP 方法

  • GET 方法：获取资源

  • POST 方法：传输实体主体

  • PUT 方法：传输文件（存在安全性问题）

  • HEAD 方法：获得报文首部

  • DELETE 方法：删除文件（存在安全性问题）

  • OPTIONS 方法：询问支持的方法

  • TRACE 方法：追踪路径（不常用）

  • CONNECT 方法：要求用隧道协议连接代理，要求在于代理服务器通信时建立隧道，主要使用 SSL（Secure Sockets Layer，安全套接层）和 TLS（Transport Layer Security，传输层安全）协议把通信内容加密后经网络隧道传输。

3、请求报文和响应报文的结构

  • 请求行：包含用于请求的方法，请求 URI 和 HTTP 版本。

  • 状态行：包含表明响应结果的状态码，原因短语和 HTTP 版本。

  • 首部字段：包含表示请求和响应的各种条件和属性的各类首部。一般有四种首部：通用首部、请求首部、响应首部、实体首部。

4、HTTP 状态码

状态码是当客户端向服务器端发送请求时，描述返回的请求结果。（常用的十四种）

  • 1XX：接收的请求正在处理

  • 2XX：成功。

  • 3XX：重定向：表明浏览器需要执行某些特殊的处理以正确的处理请求。

  • 4XX ：客户端错误，表示客户端是发生错误的原因所在。

  • 5XX ：服务器错误，表明服务器本身发生错误
